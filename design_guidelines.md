# CTI Aggregator Design Guidelines

## Design Approach
**System**: Custom Dark Dashboard Design inspired by professional security platforms (Datadog, Grafana, Splunk) with Material Design principles for data visualization consistency.

**Rationale**: Information-dense cybersecurity application requiring clarity, hierarchy, and 24/7 readability. Dark theme reduces eye strain for security analysts during extended monitoring sessions.

## Core Design Elements

### A. Typography
- **Primary Font**: Inter or Roboto via Google Fonts CDN
- **Monospace Font**: JetBrains Mono for IOCs, hashes, IP addresses, CVE IDs
- **Hierarchy**:
  - Page Titles: 28px, font-weight 600
  - Section Headers: 20px, font-weight 600
  - Card Titles: 16px, font-weight 500
  - Body Text: 14px, font-weight 400
  - Table Data: 13px, font-weight 400
  - Labels/Metadata: 12px, font-weight 500

### B. Layout System
**Spacing Units**: Tailwind units of 2, 4, 6, and 8 (p-2, m-4, gap-6, py-8)
- Component padding: p-4 to p-6
- Section spacing: py-6 to py-8
- Card gaps: gap-4 to gap-6
- Sidebar width: 240px fixed
- Top bar height: 64px fixed
- Main content: max-width container with px-6 to px-8

### C. Component Library

**KPI Cards**
- Rounded corners (rounded-lg)
- Padding: p-6
- Display large metric (32px, font-weight 700) with label below
- Icon in top-right corner (24px)
- Subtle percentage change indicator with up/down arrow

**Data Tables**
- Striped rows for readability
- Header row with font-weight 600
- Cell padding: px-4 py-3
- Hover state on rows
- Fixed header on scroll for long tables
- Action buttons (View, Details) as text links or small buttons

**Charts**
- Use Recharts library consistently
- Grid lines with low opacity for reference
- Tooltips on hover with relevant data points
- Legend positioned top-right or bottom
- Responsive sizing within card containers

**Sidebar Navigation**
- Width: 240px
- Logo/branding at top (h-16)
- Navigation items with icons (from Heroicons)
- Active state: left border accent + background highlight
- Icon size: 20px, text 14px
- Spacing: py-3 px-4 per item

**Top Bar**
- Search input with icon (w-96 max-width, centered or left-aligned)
- User profile section on right (avatar + dropdown)
- Height: 64px with centered content
- Border bottom separator

**Forms (Settings Page)**
- Section grouping with headers
- Form labels: 14px, font-weight 500
- Input fields: h-10, rounded, px-3
- Toggles/switches for boolean options
- Dropdowns for multi-option selections
- Save button: primary action style, bottom-right alignment

**Severity Badges**
- Pill-shaped (rounded-full)
- Padding: px-3 py-1
- Font-size: 12px, font-weight 600
- CVSS score mapping:
  - Critical (9.0-10.0): Distinct strong treatment
  - High (7.0-8.9): Noticeable treatment
  - Medium (4.0-6.9): Moderate treatment
  - Low (0.1-3.9): Subtle treatment

**Chat Interface**
- Left sidebar: Recent sessions list (w-64)
- Main chat area: Messages in speech bubbles
- User messages: right-aligned
- Bot messages: left-aligned with avatar/icon
- Input field at bottom with send button
- Quick query buttons: Grid of suggested queries (2-column on mobile, 3-4 on desktop)

**Pagination Controls**
- Previous/Next buttons with disabled states
- Page indicator: "Page X of Y" centered
- Compact design: flex row with gap-2

### D. Consistent Patterns
- All interactive elements have clear hover states
- Buttons use consistent sizing (h-9 for small, h-10 for medium, h-11 for large)
- Icons from Heroicons library via CDN (outline style for navigation, solid for emphasis)
- Consistent card structure across all pages
- Loading states: Skeleton screens or spinner for data-heavy sections

## Images
No hero images or marketing imagery required. This is a data-focused application dashboard. Any visual elements should be:
- Icons for threat types, severity indicators
- Small avatar placeholders for user profiles
- Chart visualizations (generated by Recharts)
- Status indicators (dots/badges) for threat intelligence source health
